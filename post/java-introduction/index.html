像 C 一样经典的编程语言

Java 是目前最为流行的编程语言，即使 Python、Go 等新兴语言风靡世界，Java 依然拥有其他语言不可比拟的巨大优势，那是它二十多年来的积淀——强大的生态，包容着互联网世界发展二十多年来种种巧妙的解决方案，它就像一本互联网图鉴，记录着互联网世界的变迁和开发者们的智慧。它是与 C 一样经典不可忽视的编程语言。

一、Java 发展历史

1981 年斯坦福研制出了图形工作站样机，取名为 SUN，即斯坦福校园网（Stanford University Network）的三个首字母。之后，几个斯坦福商学院的学生加入进来，一起创办了太阳（SUN）公司，专门生产和销售这种工作站。

1991 年，SUN 公司的 James Gosling 领导的 Java 语言开发小组成立，开始着力于绿色计划的发展，Green 是一个嵌入式程序项目，目的是为家用电子消费产品开发一个分布式代码系统。在 Green 项目最开始的时候，SUN 的工程师原本打算使用 C++ 语言进行项目开发。但是考虑到 C++ 开发的复杂性，于是基于 C++ 开发出一套独立的平台 Oak，这就是 Java 的前身。后来发现 Oak 已经被另一个语言注册，于是改名为 Java。Java 从 HotJava 浏览器之后开始进军互联网，但是由于互联网低潮带来的影响， SUN 并没有得到很好的发展，2009 年 4 月 20 日被贾甲骨文公司（Oracle）以 74 亿美元的价格收购，从此 Java 归于甲骨文旗下。

二、特性

  1. 简洁有效
    Java 是一种简介的面向对象的程序设计语言，克服了 C++ 语言中如头文件、指针、结构等难以理解的点。它更严谨、简介也足够简单。
  2. 可移植性
    Java 执行在 Java 虚拟机（Java Virtual Machine，JVM）之上，在其源代码编译之后生成字节码文件交由虚拟机来运行。因此在不用操作系统上只需要植入与操作系统匹配的虚拟机就可以执行，降低了开发复杂度。
  3. 面向对象
    Java 是一门面向对象的编程语言，有更加良好的程序结构定义。
  4. 垃圾回收
    垃圾是指无用的内存，Java 提供了垃圾回收机制（Garbage Collection，GC），利用垃圾回收机制使得开发者在编写程序时只需要考虑自身程序的合理性，而不用去关注 GC 问题，极大简化了开发难度。
  5. 引用传递
    Java 使用更加简单的引用来代替指针。指针虽然是一种高效的内存处理模型，但是需要较强的逻辑分析能力。而 Java 在设计时充分考虑到了这一点，开发者可以使用简单的引用来代替复杂的指针操作。
  6. 适合分布式计算
    Java 设计的初衷是为了更好的解决网络通信问题，所以 Java 语言十分适合分布式计算程序的开发。它提供了 Socket 开发、NIO、AIO 的支持，适用于公共网关接口（Common Gateway Interface，CGI）程序的开发。
  7. 健壮性
  8. 多线程编程支持
    提供了 JUC 多线程开发框架。
  9. 较高的安全性
    Java 的执行依赖于虚拟机，增强了安全性。
  10. 函数式编程
  11. 模块化（Module）支持
    Java 9 开始支持的功能，可以将庞大冗余的 Java 分解成一个个的模块，方便开发和部署。

三、Java 编程起步

3.1 CLASSPATH

Java 程序的执行依赖于 JVM，当用户使用 Java 命令去解释 class 字节码文件时实际上都会启动一个 JVM 进程，而在这个 JVM 进程中需要有一个明确的类加载路径，而这个类加载路径就是通过 CLASSPATH 环境属性来指派的，默认情况下 CLASSPATH 指向当前目录，即所有的 class 文件都从当前文件夹中开始查找。

3.2 Hello World

代码 hello.java ：

public class hello {
	public static void main (String[] args) {
		System.out.println("Hello World!");
	}
}

编译：

javac hello.java

执行：

java hello.class
