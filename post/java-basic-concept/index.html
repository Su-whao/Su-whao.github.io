一、注释

在程序编写的过程中，使用注释可以明确的标记出代码的作用，同时也更方便维护，因此写好注释是每一个程序员的必修课。

Java 中注释的形式有三种：

  1. 单行注释，使用 // 标记
  2. 多行注释，使用 /* …… */ 标记
  3. 文档注释，使用 /** …… */ 标记

单行注释

public class hello() {
	public static void main(String[] args) {
		// 单行注释示例
		// 打印 Hello, World 字符串
		System.out.pringln("Hello, World");
	}
}

多行注释

public class hello() {
	public static void main(String[] args) {
		/* 
		 * 单行注释示例
		 * 打印 Hello, World 字符串
		 * 常用来在写大段说明文字
		 */
		System.out.pringln("Hello, World");
	}
}

文档注释

/**
 * 打印一行字符串
 * @Date 2020/05/30
 */
public class hello() {
	public static void main(String[] args) {
		System.out.pringln("Hello, World");
	}
}

文档注释中提供了许多类似 @Date 这样的标记，如参数类型 @param，返回值类型 @return 等等，开发中常用文档注释来标记类和方法，进行说明。

这是 Java String 类中 charAt 函数的注释，作为参考。

在 IEDA 等编辑器中还可能提供其他功能，如 TODO 标记可以将每一个 TODO 标记及备注文字集合到 IDE 的一个列表中，方便整体预览。

二、标识符与关键字

标识符：

程序中变量名称为标识符。在 Java 中标识只能由字母、数字、_和 $ 符号组成，并且不能以数字开头，不能使用 Java 中的保留字。

对于标识符的定义由以下建议

  1. 尽量不要使用数字，如：i1、i2
  2. 命名应尽量有意义，使用 Student、car 等单词，而不是 a、b 等无意义的字符
  3. Java 中标识符区分大小写，Book、book、bOok 表示三个不同的标识符
  4. ‘$’ 符号由特殊的意义，不要使用

常用的标识符命名方法有三种分别是匈牙利命名法、驼峰命名法和帕斯卡命名法。匈牙利命名法要求在每个变量名前面加上若干表示数据类型的字符，例如 String 类型的数字变量可以写成 s_num‘，Windows 多采用了这种方式；驼峰命名法指混合使用大小写来构成变量的名字，将变量名中第一个单词首字母小写，其他单词首字母大写，如变量名 charAt；帕斯卡命名法在驼峰命名法的基础上将首字母大写，如 UserName，通常用在类名的定义中。除此之外，定义常量通常使用全大写，如定义常量 USERCOUNT=10。

从 Java 1.7 开始也开始支持中文变量名，如 用户名="Jake"，但是在开发中通常还是使用英文作变量名，鲜有使用中文。
